# CashFlow Manager - Railway Deployment

## Quick Deploy to Railway

[![Deploy on Railway](https://railway.app/button.svg)](https://railway.app/template/django-cashflow)

This project is ready for deployment on Railway with PostgreSQL database.

## Environment Variables Required

Set these in Railway dashboard:

- `SECRET_KEY` - Django secret key (generate with: `python -c "from django.core.management.utils import get_random_secret_key; print(get_random_secret_key())"`)
- `DATABASE_URL` - Auto-generated by Railway PostgreSQL
- `DEBUG` - Set to `False` for production
- `ALLOWED_HOSTS` - Your Railway domain (*.railway.app)

## Files for Railway Deployment

- ✅ `Procfile` - Web server configuration
- ✅ `runtime.txt` - Python version
- ✅ `railway.json` - Railway build/deploy config
- ✅ `nixpacks.toml` - Build configuration
- ✅ `requirements.txt` - Python dependencies

## Deployment Steps

1. Fork/Clone this repository
2. Create new project on [Railway](https://railway.app)
3. Connect your GitHub repository
4. Add PostgreSQL database
5. Set environment variables
6. Deploy!

See `DEPLOY_RAILWAY.md` for detailed instructions.
